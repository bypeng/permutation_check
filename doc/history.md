# 🏆 TAR Final Mission 驗收與轉播系統：版本與功能說明文件

本文件總結了 `permutation_check` 專案目錄下，各個前端介面（HTML 檔案）的版本差異、對應功能，**以及系統從初期雛型到最終正式版的開發進化歷程**。
系統的核心路由由 `Code.gs` 中的 `doGet(e)` 負責，並根據系統環境變數 **「無密碼模式 (isPasswordFree)」** 的狀態，動態派發不同版本的頁面給使用者。

---

## 1. 驗收系統 (Permutation Check)
這是給關主或參賽隊伍進行「木條排序驗證」的主要介面。

### 📍 目前正式版本
*   **`permutation_check.html` (正式密碼版)**
    *   **觸發條件**：系統設定 `isPasswordFree` 為 `false` 時載入。
    *   **功能差異**：使用者必須手動輸入正確的「正式通關密碼」（`SECRET_PASSWORD`），驗證成功後才能解鎖並獲取機密資料（如：晉級隊伍名單、詩詞文本）。適用於正式比賽，防止未授權人員提早偷看題目。

*   **`permutation_check_wo_passwd.html` (無密碼/防呆版)**
    *   **觸發條件**：系統設定 `isPasswordFree` 為 `true` 時載入。
    *   **功能差異**：針對測試情境或特殊賽制設計的防呆版本。後端會允許使用「防呆自動驗證密碼」（`DUMMY_PASSWORD`）通過驗證。前端介面通常會省略密碼輸入步驟或自動帶入密碼，讓使用者（或測試人員）一秒直達驗收操作區，省去反覆輸入密碼的麻煩。

### 🌱 開發進化歷程
*   **v1 ~ v2 (基礎雛型)**：建立基本的直書 HTML/CSS 介面與 `SortableJS` 拖曳功能。實作了基於 Unicode 內碼的待選區自動排序，方便關主在茫茫字海中快速檢索竹片。
*   **v3 ~ v4 (核心演算法與進度同步)**：導入 **LIS (最長遞增子序列) 演算法**，實現自動判斷並將正確的排列亮綠燈。加入 Google Sheets 寫入功能與防抖 (Debounce) 機制，確保拖曳操作不會頻繁觸發 API。
*   **v5 ~ Final (自動化與高可用性防護)**：整合 **Line Messaging API**，達成滿 40 片自動推播至工作群組。加入 `isPasswordFree` 切換機制，並針對併發需求於後端加入 `LockService` 寫入鎖，確保資料一致性。

## 2. 轉播台 (Live Dashboard)
這是讓未晉級隊伍、觀眾或工作人員，即時觀看各隊伍「最長遞增序列 (LIS)」拼湊進度的儀表板。

### 📍 目前正式版本
*   **`live_dashboard.html` (正式版)**
    *   **觸發條件**：帶有路由參數 `?p=live` 且 `isPasswordFree` 為 `false` 時載入。
    *   **功能差異**：常規的進度直播頁面，具備定時透過 API (`getLiveStatus`) 抓取快取資料並更新畫面的能力，應付高併發的觀看人潮。

*   **`live_dashboard_wo_passwd.html` (無密碼版)**
    *   **觸發條件**：帶有路由參數 `?p=live` 且 `isPasswordFree` 為 `true` 時載入。
    *   **功能差異**：與驗收系統連動。當系統處於防呆/無密碼模式時啟用的轉播台版本。可能移除了部分正式賽中用來遮蔽或防止破壞的觀看限制，或是針對測試環境的排版進行了簡化。

### 🌱 開發進化歷程
*   **初期版本**：簡單的定時 (Polling) 抓取 Google Sheets 資料並以列表重新渲染畫面。
*   **效能優化版**：為應對決賽 60+ 人同時觀看可能造成的 GAS 呼叫限制，後端加入 `CacheService` (5 秒快取層)，大幅降低對 Google Sheets API 的直接讀取次數，完美保護配額。
*   **UX 升級版 (Final)**：加入「依抵達順序 / 依排對片數」的 **動態排序切換開關**。並導入前端界的高階技巧 **FLIP 動畫技術 (First, Last, Invert, Play)**，讓隊伍名次發生超越或變動時，卡片會平滑順暢地交換位置，視覺體驗大幅提升。

## 3. 後台管理 (Backend Setting) & 系統核心 (Code.gs)
這是專屬大關主/主辦單位使用的控制中心與後端大腦。

### 📍 目前正式版本
*   **`backend_setting.html` (單一版本)**
    *   **觸發條件**：帶有路由參數 `?p=admin` 時載入。
    *   **功能說明**：沒有區分版本，一律必須輸入「大關主後台密碼」（`ADMIN_PASSWORD`）才能登入。
    *   **核心功能**：用來一鍵切換上述的 **「無密碼模式 (isPasswordFree)」**、**「LINE 驗收自動推播開關」**，以及**「勾選晉級決賽的隊伍名單」**（未勾選的隊伍會被系統過濾，無法進行驗收）。

### 🌱 開發進化歷程
*   **初期後端**：僅依靠手動修改 `Code.gs` 常數或直接編輯 Google Sheets 來控制比賽狀態。
*   **Webhook 導入**：為了不把 Line Token 和 Group ID 寫死，實作了 `doPost(e)` 接收 Line 群組密語 (`/取得groupID`)，自動綁定推播群組，並嚴格加入三位大關主 User ID 的權限驗證。防禦機制也加入了「智慧防連發」，避免網路延遲導致重複發送相同結果的通知。
*   **圖形化後台 (Final)**：建立專屬的 `backend_setting.html` 儀表板。讓大關主可以透過直覺的 Toggle Switch 切換無密碼模式、動態勾選「晉級決賽隊伍」，系統甚至會自動過濾未晉級隊伍，嚴防非晉級隊伍誤觸或干擾決賽。
